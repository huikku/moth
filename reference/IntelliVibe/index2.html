Vibe Coding Workflow

A structured approach to building software with Large Language Models

1

Session 1: Planning & Design

1

Brainstorm 💭

Discuss approaches, technologies, and interface ideas with the LLM

  • Explore potential solutions without code
  • Consider technical constraints
  • Sketch out interface concepts
  • Identify potential challenges
2

Create Project Checklist 📋

Develop a structured checklist to track progress and keep LLMs on task

  • Break project into logical components and features
  • Track completion status with checkboxes
  • Include dependencies between components
  • Revisit and update at the start of each session
3

Create Working Plan 🗂️

Design file structure and component hierarchy for the feature

  • Map out file organization
  • Define component relationships
  • Establish data flow patterns
  • Identify reusable components
4

Develop Interfaces 🎨

Create interface designs using kablUI notation

  • Define component structure with @COMPONENT syntax
  • Specify styling with [] tags
  • Use §SECTION for logical grouping
  • Include state variations with {state}
🔄

Session Break Point

Start a fresh LLM session here to avoid context limitations. Share checklist and interface designs.

2

Session 2: Implementation & Execution

5

Generate Scaffolding Script 🏗️

Create a Python script to build all needed files and directories

  • Generate directories based on component hierarchy
  • Create placeholder files with copyright headers
  • Add function/component templates
  • Ensure consistent file organization
6

Create Development Prompt 📝

Develop a compact, LLM-readable prompt for implementation

  • Summarize project context concisely
  • Include relevant interface designs
  • Reference file structure
  • Specify coding standards
7

Write Project Manifesto 📜

Create a guiding document to keep future LLM sessions on track

  • Define project vision and principles
  • Outline technical approach
  • Document key architecture decisions
  • Establish coding standards
8

Complete Implementation ✅

Open new LLM session to implement the code based on previous steps

  • Share manifesto and prompt with new LLM session
  • Implement code using established structure
  • Reference checklist to maintain progress
  • Update checklist with completed items
🚀

Cursor/Coding LLM Onboarding

Bring your completed planning artifacts to a coding-focused LLM tool to implement the codebase.

Key Benefits

🔄

Maintains Context

Preserves critical information across LLM sessions, preventing knowledge loss due to token limitations.

📏

Structured Code

Produces consistent, well-organized code that follows established patterns and best practices.

🧠

Progress Tracking

Prevents LLM from losing track of the project's state and completion status between sessions.

🏗️

Reusable Scaffolding

Creates project templates and structures that can be reused for similar projects in the future.

🧩

Separation of Concerns

Clearly divides planning from implementation, leading to better architecture and cleaner code.

🌐

LLM Agnostic

Works across different LLM tools and models, allowing you to use the best tool for each phase.
